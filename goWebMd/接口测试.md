
##  访问URL时,用于鉴权的userID信息如何传入

Header 头传递（最推荐，RESTful API 标准做法）

Go Web 鉴权中间件最常用的解析方式是**从 HTTP 请求头（Header）中获取 userID**（如`X-User-ID`、`Token`等自定义头），这是无状态 API 的标准设计，Apifox 中配置步骤如下：

1. 打开对应接口的编辑页面，切换到**请求头**标签页；
2. 点击「添加行」，**左侧填写服务端约定的 Header 键名**（如`X-User-ID`，必须和 Go 中间件中解析的键完全一致，中间件代码示例：**`userID := r.Header.Get("X-User-ID")`）；**
3. **右侧填写具体的 userID 值**（如`1001`、`u8923746`，根据测试场景填写有效值）；
4. 直接发送请求，Apifox 会自动将`userID`携带在请求头中，Go 中间件可正常解析鉴权。
